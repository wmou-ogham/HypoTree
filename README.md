# HypoTree 🌲

> Hypothesis + Tree —— 一個 XMind 風格的**研究問題樹狀心智圖**，專為管理研究過程中的假設、證據與推論而生。

研究的推論充滿不確定性，一般心智圖只有「節點」與「連線」，無法表達「這個假設驗證了沒？」「這條路被推翻後，下游推論還站得住腳嗎？」HypoTree 把科學方法直接畫進心智圖：每個節點都有**驗證狀態**、**實驗進度**與**證據文獻**，當上游假設被推翻時，下游會自動標記為「失去支撐」。

純前端、零後端，資料存在瀏覽器（IndexedDB）。**每個研究有專屬網址**（如 `/r/如何降低-api-延遲`），可同時管理多份研究，並可書籤或分享連結。

## 多研究與網址

| 路徑 | 說明 |
| --- | --- |
| `/` | 研究列表：建立新研究、開啟既有研究、刪除 |
| `/r/{slug}` | 單一研究編輯器；slug 依建立時的研究名稱產生 |

- 建立新研究時會依名稱產生 **slug**（中文保留、空白變 `-`），並導向專屬網址
- 每份研究的節點樹**獨立存在 IndexedDB**（以 slug 為 key）
- 工具列可點 slug 複製完整網址；顯示名稱可改，網址 slug 建立後固定
- 舊版單一 `main` 文件會在首次進入首頁時自動遷移

## 功能

### 1. 節點驗證狀態視覺化

| 狀態 | 意涵 | 視覺呈現 |
| --- | --- | --- |
| 假設中 (Hypothesis) | 尚未驗證的點子或猜測 | 黃色 / 虛線框 |
| 實驗中 (Experimenting) | 正在進行實驗驗證；**有綁定證據/文獻時自動標記** | 藍色 / 實線框 / 🧪 |
| 已證實 (Validated) | 有數據或文獻支持 | 綠色 / 實線框 |
| 已推翻 (Falsified) | 實驗失敗、邏輯不通（保留痕跡避免重複踩雷） | 灰紅 / 刪除線 / 半透明 |

### 2. 證據與文獻側邊欄

點選節點，右側面板可綁定：

- **文獻引用**：標題、日期、心得注記、DOI/ArXiv/Zotero 連結（日期僅存於 App，不匯出 Markdown）
- **實驗數據**：直接貼上圖片（存入 IndexedDB）、Code 區塊、CSV/JSON 摘要
- **實驗進度**：以滑桿標記 0–100% 完成度

### 3. 推論崩塌警告 (Inference Collapse)

若推論鏈為 `A → B → C → D`，當你把 `B` 改為「已推翻」，系統會自動將下游 `C`、`D` 標記為**懸空 (orphaned)**：節點淡化、連線變斷裂虛線，提醒你這條研究線索已經斷了。

### 4. Markdown 雙向映射

圖形端有豐富的科學狀態，匯出時自動退化為標準 Markdown 任務列表：

| 前端狀態 | Markdown 語法 |
| --- | --- |
| Hypothesis | `- [ ] 節點名稱` |
| Experimenting | `- [ ] 🧪 節點名稱` |
| Validated | `- [x] 節點名稱` |
| Falsified | `- [x] ~~節點名稱~~` |

備註、實驗進度與證據以縮排子內文呈現：

```markdown
- [x] 研究題目
  - 備注內容
  - 實驗進度: 70%
  - [快取效益文獻](https://doi.org/10.1145/3289602) 論文指出多層快取可顯著降低讀取延遲
  - [標題](https://example.com) 內容說明
```

匯入 `.md` 時會反向解析此語法重建研究樹。完整狀態（座標、附件等）請用 `.json` 匯入匯出。

## 快速鍵

| 快速鍵 | 功能 |
| --- | --- |
| `Tab` | 新增子節點（下一級；在標題/備註欄亦同） |
| `Enter` | 編輯標題 → 完成標題 → 聚焦備註（依序） |
| `Shift + Enter` | 新增同層節點 |
| `Shift + 點擊` | 多選節點（再點一次可取消選取） |
| `Space` | 收合 / 展開子樹 |
| `X` | 標記為已推翻 |
| `V` | 標記為已證實 |
| `C` | 標記為實驗中 |
| `Delete` / `Backspace` | 刪除節點（含子樹，Mac 刪除鍵可用） |
| `Cmd/Ctrl + Z` | 復原 |
| `Cmd/Ctrl + Shift + Z` | 重做 |

## 技術棧

- Vite + React 18 + TypeScript
- [@xyflow/react](https://reactflow.dev)（React Flow）畫布與節點視覺化
- Zustand 狀態管理
- Dexie（IndexedDB）本地儲存與圖片附件
- Tailwind CSS

## 開始使用

```bash
npm install
npm run dev      # 啟動開發伺服器 (http://localhost:5173)
npm run build    # 型別檢查 + 打包
npm run preview  # 預覽打包結果
```

首次開啟會載入一棵以「降低系統延遲」為主題的示範研究樹，示範各種狀態與推論崩塌效果。資料自動儲存於瀏覽器，重新整理不會遺失。
